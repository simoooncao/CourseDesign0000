这是我在课程研发工作中代码写的最多的一次。当时公司决定尝试借助腾讯和百度的AI接口来开发课程，我除了需要编写调用web API的程序之外，还需要程序来解决一些其他的问题。

我们当时的一个idea是用户上传一张自拍照，我们调用腾讯的人脸搜索接口，在我们事先储存的明星群组中给出最相似的明星照片。相对来说，从网上爬取明星照片是一件比较容易的事，但是爬取到的图片尺寸大小不一，而在展示的时候这些图片都会被拉伸到600*600的尺寸，效果很差。因此我除了需要爬取图片，还需要将图片剪裁为600*600的尺寸才能上传到明星库中。

我注意到腾讯人脸接口的返回结果中也包含了人脸的定位信息，借助这个定位信息，我以人脸区域作为最终明星库图片的中心位置，向外各拓展300像素，作为剪裁的区域。这个做法的效果还可以，算是解决了这个需求。
